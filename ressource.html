Les positions en CSS

La position relative : L'élément ayant cette position est dans le flux.

Les valeurs que l'on lui donne avec top left bottom ou right décale l'élément par rapport à lui même.


La position absolute : L'élément ayant cette position n'est pas dans le flux.

Les valeurs que l'on lui donne avec top left bottom ou right positionne l'élément depuis son bord et le bord de son plus proche parent positionné(right : 10px; positionnera l'élément absolute à 10px d'écart entre le bord droit de son parent positionné et son bord droit)


La position fixed : L'élément ayant cette position n'est pas dans le flux.

Les valeurs que l'on lui donne avec top left bottom ou right positionne l'élément depuis son bord et le bord du viewport (la zone d'affichage de votre page)(right : 10px; positionnera l'élément absolute à 10px d'écart entre le bord droit de sa page son bord

droit. Il restera donc fixer a cette position sur la zone d'affichage même si un scroll est effectué).

La position sticky difficilement expliquable : allez vous informer ici : https://developer.mozilla.org/fr/docs/Web/CSS/position en très gros résumé c'est un mélange entre la position relative et la position fixed

Propriété display sur 'div'

Propriété display sur les div

Display inline, inline-block et block, sur plusieurs div par exemple :

- display: inline -> va aligner horizontalement les div à la suite d'autre éléments, mais impossible de jouer sur le height ou le width ou le margin de ces div. (padding fonctionne)

- display: inline-block -> va aligner horizontalement les div, et possibilité de jouer sur le height ou le width margin et padding de ces div

- display: block -> va forcément passer à la ligne chaque div, et possibilité de jouer sur le height ou le width margin et padding de ces div

Exécuter des scripts PHP en local

Pour tester son code PHP sans hébergement, il y a deux solutions :


1️⃣ Utiliser une plateforme qui exécuter localement les scripts PHP : Wamp ou Laragon par exemple. Ces outils peuvent aussi héberger localement des bases de données, etc ...

Il suffit ensuite de placer les fichiers PHP dans un répertoire spécifique qui se nomme en général www


2️⃣ Télécharger PHP sur le site ci-dessous. Il suffit de désarchiver l'archive et de la placer les fichiers dans un dossier sur le disque dur. Ensuite, on rajoute le chemin d'accès au PATH.

Pour vérifier que cela fonctionne, ouvrez un terminal et entrez php, tout simplement. Pour exécuter un script, php suivi du chemin d'accès du fichier php.

Laragon

Wamp, Mamp, Xampp... Il existe une multitude de serveur,

Toutefois, Laragon est le meilleur et recommande de passer dessus,

Pourquoi me diriez-vous utiliser Laragon plutôt qu'un autre?

Voilà, car:

- Il ne bug pas une fois sur deux à l'installation

- Il possède un « mail catcher ».

- La configuration du « mail sender » est ultra-simple avec Gmail.

- Il est beaucoup plus simple a appréhender.

- Il permet de plus facilement switch de version de PHP.

- Il integre Redis, Memcached, Node.Js (w/npm et yarn), Git, MySQL (mais permet de switch entre les versions).

- Il integre des outils comme HeidiSQL mais PMA est facilement installable.

- Il intègre PostrgeSQL, MongoDB.

- Il créé automatiquement des Virtuals Hosts

- Il offre la possibilitée d'utiliser le protocole HTTPS ultra facilement

- Il permet de partager facilement ses sites

- Il intégre directement des outils comme Laravel / Symfony / WordPress

- Il est très léger, peut être portable, est indépendant, rapide, facilement réglable.

Kali Linux

Kali Linux est une distribution très connu pour être la distribution Linux sur la cybersécurité. Elle possède entre autres un noyau modifié pour les besoins des tests d'intrusion, un dépôt de programmes propres avec un nombre de programmes restreints pour en assurer la sécurité, le réseau est désactivé par défaut, etc. Tout cela fait que, des choses très simples à faire avec d'autres distributions peuvent être bien plus délicates à faire avec Kali Linux. Installer Steam par exemple ne marchera pas et risque même de casser le système vous forçant à le réinstaller.

Une distribution spécifique

Kali Linux n'a pas été conçue comme une distribution à utiliser au quotidien pour naviguer sur Internet ou faire de la bureautique. Elle n'a pas non plus été conçue pour être accessible aux débutants.


Enfin, Kali Linux possède des outils qui, mal utilisés, peuvent être destructeurs et vous mettre en danger personnellement voire légalement si vous utilisez ses outils sur un réseau pour lequel vous n'avez pas reçu d'autorisation préalable : l'accès et le maintient frauduleux dans un système de traitement automatisé de données sont punis par la loi de prison allant jusqu'à 5 ans et d'amende allant jusqu'à 150 000€.


Et les pros ?

Très peu d'hackeur éthique professionnel qui travaille dans le mileu utilisent Kali Linux. Ils préfèrent d'autres distributions comme lfs ou gentoo, sur laquelles ils installent les outils qu'ils utilisent vraiment.


Alors, quoi utiliser ?

Il est vrai que les hackeurs éthique utilisent des distributions Linux dans presque tout les cas. Toutefois, il vaut mieux se tourner (au moins pour commencer) vers des distribs plus classiques comme Ubuntu ou Debian. Pourquoi ? Vous pourrez vous habituez au terminal (les pentesteurs passent le plus claire dans le temps dans un terminal en plus de Google) et vous ne serez pas tenté par les outils sans savoir comment les utiliser.
